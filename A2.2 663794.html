A2.2 LDA y árboles de decisión¶

1. Definición del problema y origen de los datos¶

1.1 Descripción del problema¶

En esta actividad se aborda un problema de clasificación supervisada multiclase, cuyo objetivo es predecir la especie de una flor del género Iris a partir de ciertas características físicas medidas en la planta. Este tipo de problema es común dentro del área de aprendizaje automático, ya que consiste en identificar patrones dentro de un conjunto de variables numéricas que permitan distinguir correctamente entre distintas categorías.

Para este análisis se utiliza el Iris Dataset. El cual se saco del UCI Machine Learning Repository, una de las colecciones más reconocidas de datasets para investigación en aprendizaje automático.

El conjunto de datos contiene mediciones morfológicas de flores pertenecientes a tres especies diferentes:

  • Iris setosa
  • Iris versicolor
  • Iris virginica

Cada observación del dataset representa una flor individual y está descrita mediante cuatro variables numéricas:

  • Sepal length (cm): longitud del sépalo
  • Sepal width (cm): ancho del sépalo
  • Petal length (cm): longitud del pétalo
  • Petal width (cm): ancho del pétalo

Estas variables conforman el conjunto de variables predictoras (X) del modelo. Por otro lado, la variable Class representa la variable de salida (y), la cual indica la especie de la flor a la que pertenece cada observación.

El objetivo del análisis es utilizar estas mediciones físicas para construir modelos de clasificación capaces de identificar correctamente la especie de la flor. En particular, se compararán dos enfoques distintos de clasificación: Linear Discriminant Analysis (LDA) y árboles de decisión, analizando tanto su comportamiento geométrico como su desempeño predictivo.

2. Partición de los datos en entrenamiento y prueba¶

2.1 Justificación de la partición¶

Para evaluar correctamente el desempeño de los modelos de clasificación es necesario separar el conjunto de datos en dos subconjuntos distintos: datos de entrenamiento y datos de prueba. Esta separación permite medir la capacidad del modelo para generalizar a nuevas observaciones, es decir, su habilidad para realizar predicciones correctas sobre datos que no fueron utilizados durante el proceso de aprendizaje.

En este análisis se utiliza una partición 70% / 30%, donde aproximadamente el 70% de las observaciones se destinan al entrenamiento del modelo, mientras que el 30% restante se reserva para la evaluación del desempeño del modelo.

La elección de esta proporción representa un equilibrio entre dos necesidades importantes del proceso de modelado. Por un lado, el modelo requiere una cantidad suficiente de datos de entrenamiento para poder identificar adecuadamente los patrones presentes en las variables predictoras. Si el conjunto de entrenamiento fuera demasiado pequeño, el modelo podría no aprender correctamente la relación entre las variables y la clase objetivo.

Por otro lado, también es necesario contar con un conjunto de prueba lo suficientemente grande que permita realizar una evaluación confiable del modelo. Si el conjunto de prueba fuera demasiado pequeño, las métricas de desempeño podrían ser inestables o poco representativas del comportamiento real del modelo.

Además, en este análisis se utiliza una partición estratificada, lo que significa que la proporción de cada clase se mantiene aproximadamente igual tanto en el conjunto de entrenamiento como en el de prueba. Esto es especialmente importante en problemas de clasificación multiclase, ya que asegura que todas las categorías estén representadas en ambos subconjuntos, evitando sesgos en la evaluación del modelo.

3. Modelo LDA¶

3.1 Construccion del Modelo¶

El primer modelo de clasificación que se implementa en este análisis es Linear Discriminant Analysis (LDA). Este método busca encontrar combinaciones lineales de las variables predictoras que permitan maximizar la separación entre las diferentes clases.

En términos geométricos, LDA proyecta los datos originales a un nuevo espacio de menor dimensión donde las clases se separan lo mejor posible. Estas nuevas variables se conocen como funciones discriminantes.

Cuando existen tres clases, como en el dataset Iris, el modelo puede generar hasta dos funciones discriminantes (LD1 y LD2), las cuales permiten visualizar la separación entre las distintas especies de flores.

# predicciones
y_pred_lda = lda.predict(X_test)

Al aplicar la transformación del modelo Linear Discriminant Analysis (LDA) sobre los datos de entrenamiento, se obtuvo una nueva representación de los datos en un espacio de menor dimensión.

El resultado mostrado por el código indica que la nueva matriz de datos transformada tiene dimensiones (105, 2). Esto significa que las 105 observaciones del conjunto de entrenamiento fueron proyectadas a un espacio de 2 nuevas variables.

Estas nuevas variables corresponden a las funciones discriminantes generadas por el modelo, comúnmente denominadas LD1 (Linear Discriminant 1) y LD2 (Linear Discriminant 2).

En LDA, el número máximo de funciones discriminantes que se pueden generar está determinado por la fórmula:

𝑛 𝑑𝑖𝑠𝑐𝑟𝑖𝑚𝑖𝑛𝑎𝑛𝑡𝑒𝑠=𝑛 𝑐𝑙𝑎𝑠𝑒𝑠−1

En este caso, el problema de clasificación contiene tres clases (setosa, versicolor y virginica), por lo que el modelo puede generar como máximo dos funciones discriminantes. Esto coincide con el resultado obtenido en la transformación del dataset, donde se observa que los datos fueron proyectados a un espacio de dos dimensiones.

Estas funciones discriminantes representan combinaciones lineales de las variables originales, construidas de tal manera que maximicen la separación entre las distintas clases. En otras palabras, el modelo busca encontrar direcciones en el espacio de variables donde la distancia entre las medias de las clases sea lo más grande posible, mientras que la variabilidad dentro de cada clase sea lo más pequeña posible.

Esta nueva representación de los datos permitirá visualizar de forma más clara cómo se separan las distintas especies de flores en el espacio generado por LDA, lo cual será analizado mediante una gráfica en el siguiente apartado.

4 Modelo basado en Árboles de Decisión¶

4.1 Construcción del modelo¶

A diferencia de Linear Discriminant Analysis (LDA), que construye fronteras lineales entre las clases, los árboles de decisión utilizan un enfoque basado en reglas jerárquicas para realizar la clasificación.

Este modelo divide el espacio de variables mediante una serie de condiciones sobre las variables predictoras. Cada nodo del árbol representa una decisión basada en el valor de una variable, y cada división busca reducir la impureza de los datos dentro de los subconjuntos resultantes.

De esta manera, el árbol genera una estructura jerárquica donde cada camino desde la raíz hasta una hoja representa una regla de clasificación.

Una ventaja importante de los árboles de decisión es su interpretabilidad, ya que permiten identificar fácilmente qué variables tienen mayor influencia en la clasificación y cómo se construyen las decisiones del modelo.

In [18]:
from sklearn.tree import DecisionTreeClassifier

# crear modelo
tree = DecisionTreeClassifier(random_state=0)

# entrenar modelo
tree.fit(X_train, y_train)

# realizar predicciones
y_pred_tree = tree.predict(X_test)

Cada nodo del árbol representa una condición basada en una variable predictora, mientras que las ramas indican las decisiones que el modelo toma para clasificar las observaciones.

El nodo raíz, que representa la primera división del árbol, utiliza la variable petal width (cm) con la condición:

Esta primera división es especialmente importante, ya que separa de forma inmediata a la especie setosa del resto de las clases. En el lado izquierdo del árbol se observa un nodo con gini = 0, lo que indica que todas las observaciones que cumplen esa condición pertenecen a la misma clase. Esto confirma que la especie setosa posee características claramente diferenciadas en comparación con las otras especies del dataset.

En el lado derecho del árbol se encuentran las observaciones correspondientes a versicolor y virginica, las cuales requieren divisiones adicionales para poder ser separadas correctamente. En estas ramas, el árbol utiliza principalmente las variables petal width y petal length para realizar nuevas divisiones.

Por ejemplo, una de las reglas generadas por el modelo puede interpretarse de la siguiente manera:

  • Si petal width ≤ 0.8, entonces la flor pertenece a la especie setosa.

Mientras que para distinguir entre versicolor y virginica, el modelo utiliza reglas adicionales como:

  • Si petal width ≤ 1.75 y petal length ≤ 4.95, entonces la flor se clasifica como versicolor.
  • Si petal width > 1.75, entonces la flor se clasifica como virginica.

Estas reglas muestran que las variables relacionadas con los pétalos son las más relevantes para distinguir entre las especies de flores en el dataset Iris. Esto coincide con lo observado previamente en el modelo LDA, donde las variables asociadas a los pétalos también mostraron una fuerte capacidad para separar las clases.

En general, el árbol de decisión logra construir una serie de reglas claras que permiten clasificar correctamente las observaciones. Además, su estructura jerárquica facilita la interpretación del modelo, permitiendo entender de forma intuitiva cómo se toman las decisiones de clasificación.

4.5 Poda del árbol de decisión¶

Los árboles de decisión pueden crecer demasiado y adaptarse excesivamente a los datos de entrenamiento, fenómeno conocido como sobreajuste (overfitting). Cuando esto ocurre, el modelo puede capturar patrones muy específicos del conjunto de entrenamiento que no necesariamente se generalizan bien a nuevos datos.

Para controlar la complejidad del árbol se utiliza el método de poda por complejidad de costo (Cost Complexity Pruning). Este método introduce un parámetro de regularización denominado α (ccp_alpha), el cual penaliza la complejidad del árbol.

El parámetro α controla el tamaño del árbol de la siguiente manera:

  • α pequeño → árbol más grande y complejo
  • α grande → árbol más simple

El objetivo es encontrar el valor de α que proporcione el mejor equilibrio entre precisión del modelo y simplicidad de la estructura del árbol.

In [21]:
from sklearn.model_selection import StratifiedKFold, cross_val_score
import numpy as np

skf = StratifiedKFold(n_splits=4, shuffle=True, random_state=0)

alphas = np.linspace(0,0.05,200)

scores = []

for alpha in alphas:
    
    tree_temp = DecisionTreeClassifier(
        random_state=0,
        ccp_alpha=alpha
    )
    
    score = cross_val_score(
        tree_temp,
        X_train,
        y_train,
        cv=skf,
        scoring="f1_macro"
    ).mean()
    
    scores.append(score)

alpha_opt = alphas[np.argmax(scores)]

print("Alpha óptimo:", alpha_opt)

Alpha óptimo: 0.015829145728643218

Para seleccionar el valor óptimo de α se realizó una validación cruzada estratificada de 4 particiones, evaluando el desempeño del modelo mediante el F1-score macro.

A partir de este procedimiento se obtuvo el siguiente valor óptimo:

αopt=0.01583

Este valor corresponde al parámetro que maximiza el desempeño promedio del modelo durante el proceso de validación cruzada.

Esta gráfica permite identificar visualmente el punto donde el árbol alcanza un buen equilibrio entre simplicidad y capacidad predictiva, evitando tanto el sobreajuste como una simplificación excesiva del modelo.

Conclusión¶

Ambos modelos lograron identificar patrones relevantes en las variables predictoras, permitiendo distinguir entre las distintas especies de flores con un alto nivel de precisión.

El modelo LDA mostró una excelente capacidad para separar las clases mediante combinaciones lineales de las variables, obteniendo una exactitud perfecta en el conjunto de prueba y un desempeño promedio cercano al 98% en validación cruzada. La representación geométrica en el espacio de funciones discriminantes permitió visualizar claramente la separación entre las clases.

Por su parte, el modelo de árbol de decisión también presentó un desempeño muy alto y permitió generar reglas de clasificación claras e interpretables. Además, el proceso de poda mediante el parámetro ccp_alpha confirmó que el árbol original ya presentaba una complejidad adecuada para describir los patrones presentes en los datos.

En conclusión, ambos modelos resultan efectivos para la clasificación del dataset Iris. Sin embargo, debido a su mayor exactitud y a la clara separación observada en el espacio discriminante, LDA se presenta como el modelo más adecuado para este problema específico.
